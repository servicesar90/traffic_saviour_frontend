Redirect Inspector

Inputs

Target URL (required).

Outputs

  • Redirect steps count
  • Final status code
  • Final URL
  • Hop-by-hop chain table

SOP Checklist (Copy-Ready)

  1. Paste campaign URL and run scan.
  2. Confirm final status code is expected (usually 200 for landing).
  3. Check there are no unexpected redirect hops.
  4. Verify final URL matches intended money/safe path.
  5. Save screenshot of hop table for audit.
  6. If mismatch found, pause launch and escalate with evidence.

Bot Scanner

Inputs

  • URL
  • Bot profile
  • Device profile

Outputs

Status code, execution time, response summary, and scan history.

SOP Checklist (Copy-Ready)

  1. Select landing URL and choose primary bot profile.
  2. Run scan on mobile and desktop device profiles.
  3. Repeat same scan for cloaked/alternate URL.
  4. Compare status code and response differences.
  5. Export scan history CSV for record.
  6. Flag suspicious bot-only behavior to compliance lead.

IP Intelligence

Inputs

IPv4 or IPv6 address.

Outputs

  • Network/provider details
  • Location metadata
  • Detection flags
  • Export JSON and blacklist action

SOP Checklist (Copy-Ready)

  1. Paste suspicious IP from reports/logs.
  2. Run lookup and review provider/location/risk signals.
  3. Cross-check with campaign geo expectations.
  4. If high-risk, add to blacklist with reason note.
  5. If trusted internal/partner traffic, consider whitelist.
  6. Export JSON evidence and attach to incident note.

URL Builder

Inputs

Base URL + UTM source/medium/campaign/term/content + alias (optional).

Outputs

  • Built URL
  • Length warning
  • Template/history

SOP Checklist (Copy-Ready)

  1. Paste base destination URL.
  2. Fill all required UTM fields using naming standard.
  3. Validate URL length warning before publishing.
  4. Copy generated URL and test in browser.
  5. Save template for reuse by channel/team.
  6. Store final URL in campaign notes.

URL Shortener

Purpose

Create compact links without losing campaign intent.

SOP Checklist (Copy-Ready)

  1. Start from validated long URL only.
  2. Create readable alias using campaign naming standard.
  3. Generate short URL and open it in clean browser tab.
  4. Confirm redirect lands on correct final page.
  5. Run short URL through Redirect Inspector once.
  6. Publish only after QA pass is complete.

Deep Operations Notes

This section is intentionally detailed so teams can run the platform consistently across shifts. For every workflow on this page, define owner, input quality standard, expected output, and escalation condition. Keep a short run-log for each action so troubleshooting is evidence-based and reproducible.

Execution Standard

  • Validate data inputs before action.
  • Apply one change at a time and measure impact.
  • Capture screenshot or log output for audit trace.
  • If result is unexpected, rollback and isolate variable.

Quality Control

Use pre-check, action-check, and post-check. Pre-check verifies assumptions, action-check confirms behavior during execution, and post-check confirms business impact in reports. This three-layer method reduces silent failures in campaign operations.
