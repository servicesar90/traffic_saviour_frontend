Type 1: URL Parameter Integration (Client-Side)

Method

  1. Pass tracking params in destination URL.
  2. Keep parameter names consistent with tracker mapping.
  3. Verify params survive redirect chain.

Code Snippet

// Landing URL with tracking params
https://offer.example.com/?click_id={clickid}&sub_id={subid}&utm_source=google&utm_campaign=spring_sale_2026

Type 2: Postback Integration (Server-to-Server)

Method

  1. Capture and store click_id at click time.
  2. Configure partner/offer postback to send conversion back to tracker.
  3. Validate approved/rejected conversion status mapping.

Code Snippet

// Postback URL example
https://tracker.example.com/postback?click_id={clickid}&payout={payout}&status=approved&txn_id={transaction_id}
// Example status mapping
approved -> conversion_count = 1
rejected -> conversion_count = 0
pending  -> hold for later update

Type 3: Pixel/Webhook Integration

Method

  1. Fire client pixel on conversion event OR send webhook server-side.
  2. Include click_id/sub_id and event value.
  3. Deduplicate events using event_id/txn_id.

Code Snippet

// Pixel-style event (example)
window.trackConversion({
  event: "purchase",
  click_id: "{clickid}",
  value: "{payout}",
  currency: "USD"
});
// Webhook payload example
{
  "event": "conversion",
  "click_id": "{clickid}",
  "sub_id": "{subid}",
  "status": "approved",
  "payout": "12.50"
}

Type 4: Platform Preset Integration (Google/Meta/TikTok Style)

Method

  1. Use platform-specific token format in URL Builder.
  2. Attach UTM + dynamic tokens together.
  3. Verify macro replacement with test click.

Code Snippet

// Google-style example
utm_source=google&utm_medium=cpc&utm_campaign={campaignid}&utm_term={keyword}
// Meta-style example
utm_source=facebook&utm_medium=paid_social&utm_campaign={{campaign.name}}&utm_content={{ad.name}}

Integration QA SOP (Copy-Ready)

  1. Run test click from source and capture click_id.
  2. Check redirect chain keeps click_id intact.
  3. Trigger test conversion via postback/webhook/pixel.
  4. Confirm conversion appears in reports with correct payout.
  5. Save screenshots and final QA sign-off note.

Common Integration Failures

  • Parameter name mismatch between source and tracker.
  • click_id dropped during redirect.
  • Missing URL encoding for special characters.
  • Postback points to wrong environment.
  • Duplicate conversion events without dedupe key.

Deep Operations Notes

This section is intentionally detailed so teams can run the platform consistently across shifts. For every workflow on this page, define owner, input quality standard, expected output, and escalation condition. Keep a short run-log for each action so troubleshooting is evidence-based and reproducible.

Execution Standard

  • Validate data inputs before action.
  • Apply one change at a time and measure impact.
  • Capture screenshot or log output for audit trace.
  • If result is unexpected, rollback and isolate variable.

Quality Control

Use pre-check, action-check, and post-check. Pre-check verifies assumptions, action-check confirms behavior during execution, and post-check confirms business impact in reports. This three-layer method reduces silent failures in campaign operations.
